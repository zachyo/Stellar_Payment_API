# Webhook Signature Verification Guide

This guide explains how to verify webhook signatures sent by the Stellar Payment API to ensure they are authentic and haven't been tampered with.

## Overview

All webhook requests from the Stellar Payment API include two security headers:

- **`Stellar-Signature`**: HMAC-SHA256 signature of the request body
- **`Stellar-Timestamp`**: Unix timestamp (seconds) when the webhook was sent

These headers allow you to:
1. Verify the webhook came from the Stellar Payment API
2. Prevent replay attacks by validating the timestamp

## Headers

### Stellar-Signature

Format: `sha256=<hex_signature>`

Example:
```
Stellar-Signature: sha256=abcd1234ef5678...
```

The signature is generated by:
1. Taking the raw JSON request body as a string
2. Computing HMAC-SHA256 using your merchant webhook secret as the key
3. Converting the result to hexadecimal
4. Prefixing with `sha256=`

### Stellar-Timestamp

Format: Unix timestamp in seconds

Example:
```
Stellar-Timestamp: 1711612345
```

## Verification Steps

### Step 1: Extract Headers

```javascript
const signature = req.headers['stellar-signature'];
const timestamp = req.headers['stellar-timestamp'];
const rawBody = req.rawBody; // Must be the original request body as a string
```

### Step 2: Verify Signature

```javascript
import crypto from 'crypto';

function verifySignature(rawBody, signature, secret) {
  // Extract the hex signature from the header
  const [algorithm, hexSignature] = signature.split('=');
  
  if (algorithm !== 'sha256') {
    return false;
  }

  // Compute expected signature
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('hex');

  // Use timing-safe comparison to prevent timing attacks
  return crypto.timingSafeEqual(
    Buffer.from(hexSignature),
    Buffer.from(expectedSignature)
  );
}
```

### Step 3: Verify Timestamp (Prevent Replay Attacks)

```javascript
function verifyTimestamp(timestamp, toleranceSeconds = 300) {
  const webhookTime = parseInt(timestamp, 10);
  const now = Math.floor(Date.now() / 1000);
  const timeDiff = Math.abs(now - webhookTime);

  // Reject if timestamp is older than 5 minutes or in the future
  return timeDiff <= toleranceSeconds;
}
```

### Step 4: Complete Verification

```javascript
function handleWebhook(req, res) {
  const signature = req.headers['stellar-signature'];
  const timestamp = req.headers['stellar-timestamp'];
  const rawBody = req.rawBody;
  const merchantSecret = process.env.WEBHOOK_SECRET;

  // Verify signature
  if (!verifySignature(rawBody, signature, merchantSecret)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  // Verify timestamp
  if (!verifyTimestamp(timestamp)) {
    return res.status(401).json({ error: 'Timestamp outside tolerance window' });
  }

  // Webhook is authentic, process it
  const payload = JSON.parse(rawBody);
  // ... handle payment event
}
```

## Important Notes

### Raw Body Requirement

The signature is computed over the **raw request body as a string**, not the parsed JSON object. This is critical:

```javascript
// ✓ CORRECT - Use raw body
const rawBody = req.rawBody; // "{"event":"payment.confirmed",...}"
const signature = crypto.createHmac('sha256', secret).update(rawBody).digest('hex');

// ✗ WRONG - Don't re-stringify parsed JSON
const parsed = JSON.parse(req.body);
const wrongSignature = crypto.createHmac('sha256', secret).update(JSON.stringify(parsed)).digest('hex');
// This will fail because JSON.stringify may format differently than the original
```

### Express Middleware Setup

To capture the raw body before Express parses it:

```javascript
import express from 'express';

app.use(express.raw({ type: 'application/json' }));

app.post('/webhook', (req, res) => {
  const rawBody = req.body; // This is a Buffer
  const rawBodyString = rawBody.toString('utf-8');
  
  // Now verify signature using rawBodyString
  // Then parse the JSON
  const payload = JSON.parse(rawBodyString);
});
```

### Timing-Safe Comparison

Always use `timingSafeEqual` to compare signatures. This prevents timing attacks where an attacker could guess the signature byte-by-byte by measuring response times.

```javascript
// ✓ CORRECT - Timing-safe comparison
crypto.timingSafeEqual(Buffer.from(actual), Buffer.from(expected));

// ✗ WRONG - Vulnerable to timing attacks
actual === expected;
```

### Timestamp Tolerance

The default tolerance is 5 minutes (300 seconds). This allows for:
- Clock skew between servers
- Network latency
- Processing delays

You can adjust this based on your security requirements:

```javascript
// Stricter: 1 minute
verifyTimestamp(timestamp, 60);

// More lenient: 10 minutes
verifyTimestamp(timestamp, 600);
```

## Secret Rotation

During secret rotation, the API may send webhooks signed with either:
- The current secret
- The previous secret (during grace period)

Your verification code should handle both:

```javascript
function verifyWithRotation(rawBody, signature, currentSecret, previousSecret) {
  // Try current secret first
  if (verifySignature(rawBody, signature, currentSecret)) {
    return true;
  }

  // Try previous secret if available
  if (previousSecret && verifySignature(rawBody, signature, previousSecret)) {
    return true;
  }

  return false;
}
```

## Example: Node.js with Express

```javascript
import express from 'express';
import crypto from 'crypto';

const app = express();

// Capture raw body
app.use(express.raw({ type: 'application/json' }));

app.post('/webhook', (req, res) => {
  const signature = req.headers['stellar-signature'];
  const timestamp = req.headers['stellar-timestamp'];
  const rawBody = req.body.toString('utf-8');
  const secret = process.env.WEBHOOK_SECRET;

  // Verify signature
  const [algo, hexSig] = signature.split('=');
  if (algo !== 'sha256') {
    return res.status(401).json({ error: 'Invalid signature algorithm' });
  }

  const expected = crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('hex');

  try {
    if (!crypto.timingSafeEqual(Buffer.from(hexSig), Buffer.from(expected))) {
      return res.status(401).json({ error: 'Invalid signature' });
    }
  } catch {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  // Verify timestamp
  const webhookTime = parseInt(timestamp, 10);
  const now = Math.floor(Date.now() / 1000);
  if (Math.abs(now - webhookTime) > 300) {
    return res.status(401).json({ error: 'Timestamp outside tolerance' });
  }

  // Process webhook
  const payload = JSON.parse(rawBody);
  console.log('Received payment event:', payload);
  res.json({ ok: true });
});

app.listen(3000);
```

## Troubleshooting

### Signature Verification Fails

1. **Check raw body**: Ensure you're using the original request body, not a re-stringified version
2. **Check secret**: Verify you're using the correct webhook secret from your merchant settings
3. **Check encoding**: Ensure the body is UTF-8 encoded
4. **Check header format**: Signature should be `sha256=<hex>`, not just `<hex>`

### Timestamp Verification Fails

1. **Check server time**: Ensure your server's clock is synchronized (use NTP)
2. **Check tolerance**: Increase tolerance if you have high latency
3. **Check timezone**: Timestamp is in UTC seconds, not milliseconds

## Security Best Practices

1. **Store secret securely**: Never commit webhook secrets to version control
2. **Use environment variables**: Load secrets from environment variables
3. **Rotate secrets regularly**: Use the secret rotation endpoint
4. **Log verification failures**: Monitor for suspicious activity
5. **Implement rate limiting**: Protect your webhook endpoint from abuse
6. **Use HTTPS**: Always use HTTPS for webhook endpoints
7. **Validate payload**: After verification, validate the payload structure
